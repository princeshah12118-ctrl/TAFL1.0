# 🔤 Regular Expression Tool — Theory of Computation Mini Project

A fully browser-based **Regular Expression Lab** built for Theory of Computation (TAFL) courses. Enter any regular expression using an easy palette, generate all accepted strings, visualize the DFA, and verify equivalence between two expressions — all powered by a real NFA engine with zero dependencies.

---

## ✨ Features

- **📐 TAFL Mode** — `+` acts as union (OR), matching formal automata theory notation: `(a+b)*` = all strings over {a, b}
- **💻 Programming Mode** — `+` acts as one-or-more, standard regex behavior
- **📋 String Set Generator** — generates all strings accepted by the R.E. up to a chosen max length, grouped by length
- **🔷 DFA Visualizer** — converts R.E. → NFA → DFA via subset construction and renders an interactive SVG diagram
- **⚖️ Equivalence Verifier** — checks if two regular expressions define the same language
- **🔬 String Tester** — test whether a specific string is accepted or rejected
- **🕑 History** — browse and reload previously entered expressions
- **📖 Reference Guide** — symbol reference with TAFL and programming notation explained

## 🚀 How to Run

No installation or server required. Just open the HTML file in any modern browser:

```bash
# Option 1: Double-click
open index.html

# Option 2: Use a local server (recommended)
npx serve .
# or
python -m http.server 8000
```

Then visit `http://localhost:8000` in your browser.

## 🧠 How It Works

### Parser
The R.E. is parsed into an **Abstract Syntax Tree (AST)** using a recursive descent parser supporting:

| Symbol | TAFL Mode | Programming Mode |
|--------|-----------|-----------------|
| `*` | Kleene star (zero or more) | Kleene star |
| `+` | **Union** (a OR b) | One or more |
| `\|` | Union | Union |
| `?` | Optional (zero or one) | Optional |
| `( )` | Grouping | Grouping |
| `ε` | Empty string | Empty string |

### NFA Engine
The AST is compiled into an **NFA** using Thompson's construction. No JavaScript `RegExp` is used — all matching is done by simulating ε-closures over NFA states.

### DFA Construction
DFAs are built from the NFA using the **Subset Construction** algorithm and rendered as SVG with a force-repel layout.

## 📐 TAFL Notation Examples

| Expression | Meaning | Example strings |
|------------|---------|-----------------|
| `(a+b)*` | All strings over {a, b} | ε, a, b, aa, ab, ba, bb, … |
| `(a+b)*abb` | Strings ending in `abb` | abb, aabb, babb, … |
| `a*b*` | Any a's followed by any b's | ε, a, b, aa, ab, bb, … |
| `(0+1)*1` | Binary strings ending in 1 | 1, 01, 11, 001, … |

## 📌 Project Objective

> The input will be a regular expression. The tool provides an easy palette to write various regular expressions and after writing the regular expression, it shows the **set of strings generated by the R.E.** It also includes a module to **verify the equivalence of regular expressions.**
